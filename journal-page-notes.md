# Journal Page Notes

These notes are for the project only and should not appear on the public journal page.

## Public structure

- The journal has two categories only:
- `GAME DEV / SOFTWARE`
- `VIEWS / DIARY / CASUAL NOTES`
- Each category panel uses a reading list on the left and a featured read on the right.

## Content direction

- The first category is for game development notes, software systems thinking, and build logs.
- The second category is for more casual writing: opinions, religion, politics, animal care, technology, reviews, and diary-style notes.
- The page should stay minimal and professional in public.

## Growth notes

- The structure should support both longer essays and shorter status-like notes.
- New entries should fit into the existing layout without redesigning the page.
- The reading list can grow into a searchable archive later without changing the overall panel layout.

## Sharing note

- Each article currently supports lightweight sharing.
- On supported devices, the native share sheet can open.
- Otherwise, visitors can copy a direct link to the article.
